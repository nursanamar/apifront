login

Setiap request yang dikirm harus menyertakan token sebagai autentikasi. Token bisa di dapatkan mengunakan login endpoint POST /login

Request
POST /login HTTP/1.1
        Host: <your-host>
        Content-Type: application/json

        {"user":<username>,"pass":<password>}

Header Content-Type: application/json harus selalu disertakan

Response

Jika login berhasil anda akan mendapatkan respon seperti ini

HTTP 200 OK
        Access-Control-Allow-Origin: *
        X-Message: ApiBuilder/1.0
        Server: ApiBuilder
        Content-Type: application/json

        {
            "status": "ok",
            "desc": "Login succes",
            "data": {
                "token": <your token>,
                "user": {
                    "id": <id>,
                    "name": <User full name>
                }
            }
        }

Anda bisa mengunakan data.token sebagai token anda unruk request berikutnya

Tapi Jika login gagal respon yang akan anda dapatkan adalah

HTTP 200 OK
        Access-Control-Allow-Origin: *
        X-Message: ApiBuilder/1.0
        Server: ApiBuilder
        Content-Type: application/json

        {
            "status": "failed",
            "desc": "login failed, please check your pass"
        }

Itu artinya anda harus memeriksa user dan pass yang anda kirim

Ambil data

Untuk megambil data sementara hanya tersedia fitur ambil semua dan ambil berdasarkan ID data

Setiap request selain login harus menyertakan Header Authorization: Bearer <your-token> yang berisi token yang di dapat pada saat login, jika tidak maka respon akan mengembalikan status header 401

        Content-Type: application/json

        Server: ApiBuilder

            "status": "ok",

Tambah data

Request
POST /<nama-tabel> HTTP/1.1
        Host: <your-host>
        Content-Type: application/json
        Authorization: Bearer <your-token>

        [
          {
            "<field-name>": "<value>",
            "<field-name>": "<value>",
            "<field-name>": "<value>"
          }
        ]
Respons
HTTP 201 CREATED
        Access-Control-Allow-Origin: *
        X-Message: ApiBuilder/1.0
        Server: ApiBuilder
        Content-Type: application/json

        {
            "status": "ok",
            "effect": 1,
            "data": [
            {
              "id":"<value>"
              "<field-name>": "<value>",
              "<field-name>": "<value>",
              "<field-name>": "<value>"
            }
            ]
        }

Edit data

Request
PUT /<nama-tabel>/<id> HTTP/1.1
        Host: <your-host>
        Content-Type: application/json
        Authorization: Bearer <your-token>

          {
            "<field-name>": "<value>",
            "<field-name>": "<value>",
            "<field-name>": "<value>"
          }

Anda bisa mengirim kolom dalam bentuk lengkap seperti di atas atau cukup kolom yang ingin di ubah nilainya

Respons
HTTP 200 OK
        Access-Control-Allow-Origin: *
        X-Message: ApiBuilder/1.0
        Server: ApiBuilder
        Content-Type: application/json

        {
            "status": "ok",
            "effect": 1,
            "data": [
            {
              "id":"<value>"
              "<field-name>": "<value>",
              "<field-name>": "<value>",
              "<field-name>": "<value>"
            }
            ]
        }

Respon akan mengembalikan data dalam bentuk lengkap yang telah diubah

Hapus data

Request
DELETE /<nama-tabel>/<id> HTTP/1.1
        Host: <your-host>
        Content-Type: application/json
        Authorization: Bearer <your-token>
Respons

Respon hanya akan mengembalikan status code 204
